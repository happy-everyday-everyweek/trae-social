# 权限与前台服务

本文档说明 Trae Social Android 应用的权限清单、运行时申请、前台服务类型与开机自启机制。

## 权限清单

| 权限 | 用途 | 申请方式 |
| --- | --- | --- |
| INTERNET | 网络访问（LLM 调用） | 普通 |
| ACCESS_NETWORK_STATE | 检测网络状态 | 普通 |
| CAMERA | 相机取景（feature-publish） | 运行时 |
| READ_MEDIA_IMAGES | 读取相册图片（API 33+ 细粒度媒体权限，EditorModeContent PickVisualMedia） | 运行时 |
| FOREGROUND_SERVICE | 前台服务基础权限 | 普通 |
| FOREGROUND_SERVICE_SPECIAL_USE | API 34+ 必须声明具体前台服务类型（specialUse） | 普通 |
| POST_NOTIFICATIONS | 前台服务通知（API 33+ 运行时申请） | 运行时 |
| RECEIVE_BOOT_COMPLETED | 开机自启（BootReceiver） | 普通 |
| WAKE_LOCK | 唤醒锁 | 普通 |

> 注：
> - `READ_EXTERNAL_STORAGE` 已移除（IMPL-46：`EditorModeContent` 用 `PickVisualMedia` 系统代理，无需存储权限）。
> - `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` 已移除（IMPL-46：Play 政策敏感且未运行时申请）。
> - `uses-feature android:name="android.hardware.camera" android:required="false"`。

## 运行时权限申请

### POST_NOTIFICATIONS

- `MainActivity` 的 `SocialApp` composable 内，`LaunchedEffect(Unit)` 中 API 33+ 检查 `POST_NOTIFICATIONS` 是否已授予。
- 未授予则 `rememberLauncherForActivityResult(RequestPermission()).launch(Manifest.permission.POST_NOTIFICATIONS)`。
- 回调为空不阻塞流程。

### CAMERA

- `feature-publish` 的 `CameraModeContent` 用 accompanist `rememberPermissionState(Manifest.permission.CAMERA)`。
- 未授予显示 `PermissionRequestCard`（前往设置/再次请求）。

### READ_MEDIA_IMAGES

- `feature-publish` 的 `EditorModeContent` 用 `PickVisualMedia` launcher（系统代理，无需显式申请）。

## 前台服务（SchedulerForegroundService）

- `foregroundServiceType="specialUse"`（IMPL-17：Android 14+ 避免 `dataSync` 6 小时配额限制，且本服务不做数据同步）。
- `<property android:name="android.app.PROPERTY_SPECIAL_USE_FGS_SUBTYPE" android:value="social_ai_scheduler_keepalive" />`（声明 specialUse 子类型）。
- 通知渠道 `scheduler` `IMPORTANCE_LOW`，常驻通知"Trae Social 运行中 / 伙伴们正在活跃中，保持连接以接收最新动态"。
- `onCreate`：`ServiceCompat.startForeground(this, NOTIFICATION_ID=1001, buildNotification(), getServiceType())`（P0 修复：三参数重载兼容 `minSdk=26`，`startForeground(int,Notification,int)` 自 API 29 才引入）。
- `onStartCommand`：`schedulePendingInteractions()` 入队 `PendingInteractionWorker` 15min KEEP，返回 `START_STICKY`（被杀后系统尝试重建）。
- `getServiceType()`：API 34+ 返回 `FOREGROUND_SERVICE_TYPE_SPECIAL_USE`，否则 `0`。
- 启动方：`SchedulerInitializer.initialize` 或 `BootReceiver`（经 `SchedulerInitializerWorker`）。
- 启动时机：`MainActivity.onCreate`（前台上下文，避免 Android 12+ 后台启动 `ForegroundServiceStartNotAllowedException`）。

## 开机自启（BootReceiver）

- 监听 `ACTION_BOOT_COMPLETED` + `ACTION_LOCKED_BOOT_COMPLETED`（Direct Boot 模式，设备解锁前也能唤醒调度）。
- `app/AndroidManifest.xml` 声明 `exported="true"` `directBootAware="false"`，intent-filter 注册两个 action（`core-scheduler/AndroidManifest.xml` 仅注册 `BOOT_COMPLETED`，app 的覆盖合并才是权威）。
- IMPL-18：不用 `Handler.postDelayed`（进程可能在 30 秒回调前被杀），改用 `OneTimeWorkRequestBuilder<SchedulerInitializerWorker>().setInitialDelay(30, SECONDS).addTag(WorkerTags.BOOT_INIT)`，`enqueueUniqueWork(BOOT_INIT, KEEP)`。
  - 延迟 30 秒避免开机资源争抢。
  - 进程被杀 WorkManager 仍能重放。
- `SchedulerInitializerWorker`：`@HiltWorker` `CoroutineWorker`，`doWork` 调 `SchedulerInitializer.initialize(applicationContext)`。

## WorkManager 初始化

- 移除 `androidx.startup` 默认 `WorkManagerInitializer`（AndroidManifest `InitializationProvider` 内 `tools:node="remove"`）。
- `SocialApp` 实现 `Configuration.Provider`，`workManagerConfiguration` getter 返回 `Configuration.Builder().setWorkerFactory(workerFactory).setMinimumLoggingLevel(Log.INFO).build()`。
- `HiltWorkerFactory` 使 `@HiltWorker` + `@AssistedInject` 的 Worker 可注入。

## FileProvider

- authority = `${applicationId}.fileprovider`（debug 为 `com.trae.social.debug.fileprovider`）。
- `exported=false`，`grantUriPermissions=true`。
- `file_paths.xml`：cache-path `images/` 与 `shared/`、files-path `.`、external-files-path `.`、external-cache-path `.`。
- 用途：相机拍照临时输出（`cacheDir/capture/`）、编辑后图片（`cacheDir/edit/`）。
