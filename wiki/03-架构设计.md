# 03 架构设计

## 模块依赖图

```
                              +---------+
                              |   app   |  Application / MainActivity / 导航 / DI 装配
                              +----+----+
        +----------+----------+-----+----+----------+----------+
        v          v          v     v    v          v          v
   feature-feed  timeline  profile  publish  onboarding   (feature-profile)
        |          |          |       |         |          |
        +----------+-----+----+-------+---------+----------+
                          |
                          v
                  +-------+-------+
                  |               |
          +-------+-------+   +---+-----------+
          v               v   v               v
   core-designsystem  core-data  core-llm  core-profiling
                          |        ^          |     ^
                          |        |          v     |
                          +--------+------+  (依赖 core-data / core-llm)
                                   |     |
                                   v     |
                            core-scheduler (反向调用 core-profiling.ProfileVersionStore
                                            与 core-llm.UserProfilePromptBuilder)
```

Note：`core-profiling` 不依赖 `core-scheduler`；第 3 层 LLM 画像的周期触发由 `core-scheduler` 中的 `UserProfileWorker` 反向调用 `core-profiling` 的 `ProfileVersionStore` 完成。`feature-profile` 同时依赖 `core-scheduler`（手动触发 Worker）与 `core-profiling`（ProfileChat / DevOptions 画像调试）。

## 依赖规则

- app 依赖所有 feature 与 core 模块，负责 DI 装配与导航
- feature 模块仅依赖 core 模块，不直接依赖其他 feature 模块（强制解耦）
- core-scheduler 依赖 core-data（DAO/Repository）与 core-llm（LlmClient）
- core-llm 通过 `api(project(":core-data"))` 传递依赖，复用 LlmProvider / AiActivityLevel 枚举定义，消除重复
- core-profiling 依赖 core-data（Entity/DAO/Repository/domain model）与 core-llm（LlmClient、PromptBuilder），不依赖 core-scheduler；不定义 `@Database`/`@Dao`，所有持久化通过 core-data 提供
- core-scheduler 的 `UserProfileWorker` 反向调用 core-profiling 的 `ProfileVersionStore` 与 core-llm 的 `UserProfilePromptBuilder` 完成第 3 层 LLM 画像周期生成
- 跨模块共享类型定义在最底层的 core-data（如 LlmProvider、UserActionType、UserProfileSnapshot、FeedbackAction）

## 单 Activity 架构

MainActivity 是唯一 Activity（`@AndroidEntryPoint`），承载整个 Compose 树。采用双层 NavHost：

- 顶层 NavHost 区分 onboarding/main 起始路由（依据 `ConfigRepository.isOnboardingCompleted()` 异步判定，加载中渲染空 Box 避免起始路由闪烁）
- main 内嵌 NavHost 承载三页 Tab 路由 + 全屏路由
- 两层 `rememberNavController` 独立

## 路由清单

| 路由常量 | 值 | 类型 | 说明 |
| --- | --- | --- | --- |
| ONBOARDING | "onboarding" | 顶层 | 引导流程 |
| MAIN | "main" | 顶层 | 主框架（Tab 容器） |
| FEED | "feed" | 主 Tab | 首页信息流 |
| TIMELINE | "timeline" | 主 Tab | 时间线 |
| PROFILE | "profile" | 主 Tab | 我的 |
| SETTINGS | "settings" | 全屏 | 设置 |
| API_KEY | "apikey" | 全屏 | API Key 管理 |
| DEV_OPTIONS | "devoptions" | 全屏 | 开发者选项 |
| PUBLISH | "publish" | 全屏（底部滑入） | 发布 |
| FOLLOW_LIST | "followlist/{type}" | 全屏（带参数） | 关注/粉丝列表 |

路由常量集中在 `AppRoutes` object，工具函数 `followList(type)` 构造带枚举名的路由。Tab 切换使用 `popUpTo(startDestination){saveState=true}` + `launchSingleTop` + `restoreState` 标准状态保持模式。

## 三页主架构

- 首页信息流（feature-feed，Paging 3 分页）
- 时间线（feature-timeline，朋友圈式图片网格）
- 我的（feature-profile，账号/推文/设置）

底部 `SocialBottomBar` 使用 `GlassBlurContainer` 磨砂玻璃，含 3 个 Tab + 圆形发布按钮（systemBlue，56dp，按压缩放 0.92 反馈）。滚动状态通过 `provideIsScrolling` CompositionLocal 传递给 `GlassBlurContainer`，滚动中模糊半径减半以提升性能。

## DI 装配策略

跨模块桥接绑定集中在 app 模块，避免 DuplicateBindings：

- `ColdStartFiller`（feature-onboarding 接口，`AppColdStartFiller` 实现）
- `AssetProvider`（core-data 接口，`AssetProviderImpl` 实现，因 assets 由 app 打包）
- `LlmConfigProvider`（core-llm 接口，`AppLlmConfigProvider` 实现，桥接 `ConfigRepository`）
- `LlmCacheInvalidator`（core-data 接口，lambda 调用 `LlmProviderRegistry.invalidateCache`）

WorkManager 通过 `SocialApp` 实现 `Configuration.Provider` 接入 `HiltWorkerFactory`，Manifest 中 `tools:node="remove"` 移除默认 `WorkManagerInitializer`。

## 分层职责一句话

| 模块 | 一句话职责 |
| --- | --- |
| app | Application / MainActivity / 导航 / DI 装配 |
| core-designsystem | 主题、颜色、公共 Composable、磨砂玻璃容器 |
| core-data | Room 数据库、DAO、Repository、Entity、TypeConverters |
| core-llm | 多 LLM 提供商客户端、流式/非流式、Prompt 构造 |
| core-scheduler | AI 推文/互动/人设更新/画像周期生成调度，WorkManager 与活跃窗 |
| core-profiling | 用户行为建模 5 层（捕获 / 基础分析 / LLM 深度画像 / 画像反哺 / 用户反馈智能体） |
| feature-feed | 首页信息流，Paging 3 分页 |
| feature-timeline | 朋友圈式时间线图片网格 |
| feature-profile | 个人主页、推文/媒体/点赞 Tab、设置入口、画像调校对话与调试开关 |
| feature-publish | 相机发布、多图、裁剪、滤镜 |
| feature-onboarding | 首次启动引导流程、LLM 提供商选择与连接测试 |
