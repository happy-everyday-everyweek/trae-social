# 开发指南

本文档说明 Trae Social Android 项目的开发环境、代码规范、Hilt DI 约定、测试、提交规范、PR 流程。

## 开发环境

- JDK 17；Android SDK 34（compileSdk），minSdk 26；Gradle 8.14+（项目含 wrapper）；Android Studio Koala+ 或 IntelliJ IDEA。

## 开发流程

1. Fork 仓库并克隆。
2. 基于 `main` 创建特性分支（`feature`/`fix`/`refactor`/`docs`/`chore`）。
3. 编码确保编译与测试通过。
4. 提交（Conventional Commits）。
5. 推送 Fork 创建 PR。

## 分支命名

- `feature/` 新功能。
- `fix/` Bug 修复。
- `refactor/` 重构。
- `docs/` 文档。
- `chore/` 构建/工具/杂项。

## 代码规范

### Kotlin

- 遵循 Kotlin Coding Conventions。
- 优先 `val` 避免可变状态。
- 避免 `!!` 强解包，用 `?.` 或 `requireNotNull`/`checkNotNull`。
- 公共 API 添加 KDoc。
- 代码注释使用中文（与项目风格一致）。

### Compose

- 状态提升到父 Composable。
- `remember`/`rememberSaveable` 管理局部状态。
- 避免 Composable 体内创建昂贵对象，用 `remember` 缓存。
- 为列表项提供稳定 key。

### 架构

- feature 模块不直接依赖其他 feature，通过 `core-*` 通信。
- 跨模块共享类型定义在最底层 core（如 `LlmProvider` 在 `core-data`）。
- 新增 Room Entity/DAO 需递增数据库版本并编写 Migration，导出 schema JSON。

## Hilt DI 约定

- `@Module` + `@InstallIn(SingletonComponent::class)` 暴露绑定。
- 避免 DuplicateBindings：同一类型仅在唯一模块中提供。
- 使用 `@Qualifier` 区分同一类型不同绑定（如 `@SecurePreferences`、`@FeedImageLoader`、`@ProfileImageLoader`、`@GalleryJson`、`@Named("openai"/"anthropic"/"gemini")`）。
- 跨模块桥接绑定集中在 app 模块（`ColdStartFiller`/`AssetProvider`/`EndpointConfigProvider`）。
- `@HiltWorker` + `@AssistedInject` 用于 Worker，需 `SocialApp` 实现 `Configuration.Provider` 接入 `HiltWorkerFactory`。
- object 类用 `@EntryPoint` `@InstallIn` 接入依赖（如 `SchedulerInitializer`）。

## 测试

```
# 单元测试
./gradlew test

# Android 测试（需设备/模拟器）
./gradlew connectedAndroidTest
```

- 已集成 MockK、Turbine、MockWebServer、kotlinx-coroutines-test、room-testing、work-testing。
- core-llm 测试覆盖：`CommentPromptBuilderTest` / `PersonaUpdatePromptBuilderTest` / `PromptUtilsTest` / `TweetPostProcessorTest` / `TweetPromptBuilderTest` / `DefaultRulesetEngineTest`（#151 后降级链与 429 转换）/ `SdkExceptionClassifierTest`（#308 反射 `statusCode()`）。
- core-scheduler 测试：`ScheduleRuleResolverMissedWindowsTest`/`DailyQuotaCheckerTest`/`SchedulerRateLimiterTest` 等。
- 新增功能或修复 Bug 建议添加对应测试。

## 提交规范

Conventional Commits 格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

- type：`feat`/`fix`/`refactor`/`docs`/`chore`/`test`/`build`。
- scope：模块名（`scheduler`/`feed`/`onboarding` 等），可选。
- subject：简短描述（祈使句，不加句号）。
- body：详细说明（可选）。
- footer：关联 issue 或 breaking change（可选）。
- 示例：

```
feat(scheduler): 人设更新 Worker 支持档位缩放周期

PersonaUpdateWorker 周期按 AiActivityLevel 缩放（LOW=14天/MEDIUM=7天/HIGH=3天），
批次大小按档位 10/20/40。
```

## 构建

```
# 编译检查
./gradlew compileDebugKotlin

# Debug APK
./gradlew :app:assembleDebug

# Release APK（需配置 keystore.properties）
./gradlew :app:assembleRelease
```

## PR 检查清单

- 代码通过 `compileDebugKotlin` 编译。
- 提交信息符合 Conventional Commits。
- 新增公共 API 有 KDoc 注释。
- 不引入 `!!` 强解包。
- 不引入硬编码密钥或 API Key。
- feature 模块不直接依赖其他 feature 模块。
- 新增 Room Entity/DAO 已递增版本并编写 Migration + 导出 schema JSON。
- 不在代码/项目中使用任何表情包。
