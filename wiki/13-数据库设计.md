# 数据库设计

Room 数据库 `social.db`，当前版本 8，17 张表，14 个 DAO，7 条迁移。schema JSON 导出至 `core-data/schemas/`。

## 数据库概览

- `AppDatabase` `@Database(version=8, entities=[17个], exportSchema=true)` + `@TypeConverters(Converters)`。
- 17 张表：`accounts`、`account_active_hours`、`tweets`、`interactions`、`follow_relations`、`persona_dynamic_fields`、`user_configs`、`scheduler_logs`、`image_usages`、`comments`、`user_action_events`、`user_profile_snapshots`、`user_profile_versions`、`user_profile_overrides`、`user_profile_feedback`、`user_profile_rollbacks`、`llm_endpoints`（v8 新增，#151）。
- 14 个 DAO：`accountDao`/`tweetDao`/`interactionDao`/`followRelationDao`/`userConfigDao`/`schedulerLogDao`/`imageUsageDao`/`commentDao`/`userActionDao`/`userProfileDao`/`userProfileOverrideDao`/`userProfileFeedbackDao`/`userProfileRollbackDao`/`llmEndpointDao`（v8 新增）。
- `fallbackToDestructiveMigrationOnDowngrade()`（仅降级破坏性重建，升级走显式 Migration）。

### comments

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | TEXT | 评论 ID | PRIMARY KEY |
| tweetId | TEXT | 关联推文 | FK -> tweets(id) ON DELETE CASCADE |
| authorId | TEXT | 评论者账号 | FK -> accounts(id) ON DELETE CASCADE |
| content | TEXT | 评论正文 | NOT NULL |

v6 新建，持久化全部评论（用户评论 + AI 评论），供评论弹层加载展示。
LIKE/RETWEET 互动存储在 `interactions` 表，不在 `comments` 表。

### user_action_events

v7 新建（#146），用户行为建模第 1 层捕获层的事件原始流。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | TEXT (UUID) | 事件 ID | PRIMARY KEY |
| type | TEXT | UserActionType 枚举 name | NOT NULL |
| screen | TEXT | 事件发生页面 | NOT NULL |
| targetId | TEXT | 目标对象 ID（如 tweetId） | |
| targetKind | TEXT | 目标对象类型 | |
| extra | TEXT (JSON) | 附加数据（如 durationMs、textTopic 等） | |
| durationMs | INTEGER | 停留时长（仅 TWEET_DWELL 等） | |
| occurredAt | INTEGER | 事件发生时间 epoch millis | NOT NULL |
| session | TEXT | 会话 ID（SessionManager 生成） | NOT NULL |

4 个索引：

- `index_user_action_events_occurredAt`（按时间扫描事件窗口）
- `index_user_action_events_type_occurredAt`（按类型 + 时间分析）
- `index_user_action_events_session`（按会话聚合）
- `index_user_action_events_targetId_type`（按目标去重 / 反哺效果回测）

无外键。`EventCleanupWorker` 按 `getEventTtlDays`（默认 14 天）周期 `deleteBefore(cutoff)` 清理过期事件。

### user_profile_snapshots

v7 新建（#146），基础分析层（第 2 层）输出的画像快照。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INTEGER | 自增 ID | PRIMARY KEY AUTOINCREMENT |
| payload | TEXT (JSON) | `UserProfileSnapshot` 序列化 | NOT NULL |
| eventWindowStart | INTEGER | 事件窗口起 epoch millis | NOT NULL |
| eventWindowEnd | INTEGER | 事件窗口止 epoch millis | NOT NULL |
| computedAt | INTEGER | 计算时间 epoch millis | NOT NULL |
| source | TEXT | `INCREMENTAL` / `FULL_RECOMPUTE` / `COLD_START_SEEDING` | NOT NULL |

索引：`index_user_profile_snapshots_computedAt`。无外键。

### user_profile_versions

v7 新建（#146），LLM 深度画像层（第 3 层）输出的版本，同一时刻至多一个 `isActive=1`。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INTEGER | 版本 ID | PRIMARY KEY AUTOINCREMENT |
| payload | TEXT (JSON) | LLM 输出结构化字段 | NOT NULL |
| narrative | TEXT | 画像叙事文本（100-300 字） | NOT NULL |
| modelProvider | TEXT | 生成该版本的 LLM 提供商 | NOT NULL |
| promptHash | TEXT | Prompt 模板哈希（版本演进追踪） | NOT NULL |
| inputFingerprint | TEXT | 输入指纹（缓存复用判定） | NOT NULL |
| snapshotId | INTEGER | 关联基础快照 ID | |
| rollbackFrom | INTEGER | 回滚来源版本 ID | |
| isActive | INTEGER | 是否当前激活版本（0/1） | NOT NULL DEFAULT 0 |
| createdAt | INTEGER | 创建时间 epoch millis | NOT NULL |

4 个索引：

- `index_user_profile_versions_snapshotId`（关联快照查询）
- `index_user_profile_versions_createdAt`（版本时间线）
- `index_user_profile_versions_inputFingerprint`（指纹缓存命中判定）
- `index_user_profile_versions_isActive`（快速定位当前版本）

无外键。`ProfileVersionStore.insertAndActivate` 用 `@Transaction` 保证"旧 active 置 0 + 新版本插入置 1"原子，避免双 active；`MAX_PROFILE_VERSIONS=100` 超限时 `deleteOldestInactive` 删除最旧非激活版本。

### user_profile_overrides

v7 新建（#146），用户反馈智能体（第 5 层）与画像反哺层（第 4 层）写入的显式覆盖。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INTEGER | 覆盖 ID | PRIMARY KEY AUTOINCREMENT |
| type | TEXT | 覆盖类型（如 theme / preference / scenario / activeHours） | NOT NULL |
| key | TEXT | 覆盖键（具体主题/场景编号等） | NOT NULL |
| value | TEXT | 覆盖值 | NOT NULL |
| reason | TEXT | 覆盖原因 | NOT NULL |
| createdAt | INTEGER | 创建时间 epoch millis | NOT NULL |
| source | TEXT | `SOURCE_FEEDBACK_AGENT` 等 | NOT NULL |
| superseded | INTEGER | 是否被新覆盖取代（软删除，0/1） | NOT NULL DEFAULT 0 |

2 个索引：

- `index_user_profile_overrides_type_key`（按类型 + 键定位同链覆盖）
- `index_user_profile_overrides_superseded`（快速过滤有效覆盖）

无外键。新覆盖写入时同 `(type, key)` 的旧覆盖标记 `superseded=1`（不删除，形成审计链）；`resetAll()` 才物理删除全部覆盖。

### user_profile_feedback

v7 新建（#146），用户反馈智能体对话历史。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INTEGER | 消息 ID | PRIMARY KEY AUTOINCREMENT |
| role | TEXT | `USER` / `ASSISTANT` | NOT NULL |
| content | TEXT | 消息文本 | NOT NULL |
| appliedActions | TEXT (JSON) | ASSISTANT 应用的 FeedbackAction 列表 | |
| rollbackPreviews | TEXT (JSON) | ASSISTANT 生成的回滚预览 | |
| createdAt | INTEGER | 创建时间 epoch millis | NOT NULL |

索引：`index_user_profile_feedback_createdAt`。无外键。`UserProfileFeedbackDao.recent(limit)` 按时间倒序取最近 N 条作为智能体上下文。

### user_profile_rollbacks

v7 新建（#146），版本回滚审计记录。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INTEGER | 回滚记录 ID | PRIMARY KEY AUTOINCREMENT |
| fromVersionId | INTEGER | 回滚前激活版本 ID | NOT NULL |
| toVersionId | INTEGER | 回滚后激活版本 ID | NOT NULL |
| reason | TEXT | 回滚原因 | NOT NULL |
| appliedAt | INTEGER | 回滚应用时间 epoch millis | NOT NULL |

索引：`index_user_profile_rollbacks_appliedAt`。无外键。`ProfileVersionStore.applyRollback` 激活旧版本但不删除新版本，同时写一条审计记录到本表。

### llm_endpoints

v8 新建（#151），LLM 网关层多端点配置表。取代旧 EncryptedSharedPreferences 按 provider 槽位寻址的配置抽象。

| 列名 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | TEXT (UUID) | 端点唯一标识 | PRIMARY KEY |
| displayName | TEXT | UI 展示名（如 "OpenAI 主端点"、"本地 Ollama"） | |
| protocol | TEXT | LlmProtocol id（`openai_compat` / `anthropic_compat`） | NOT NULL |
| baseUrl | TEXT | Base URL（如 `https://api.openai.com/v1/`） | NOT NULL |
| model | TEXT | 模型名（如 `gpt-4o-mini`） | |
| capabilities | TEXT | ModelCapability 逗号分隔字符串（如 `TEXT,JSON_MODE_NATIVE,STREAMING`） | |
| orderIndex | INTEGER | 全局排序，0 = 主模型 | NOT NULL |
| createdAt | INTEGER | 创建时间 epoch millis | NOT NULL |
| updatedAt | INTEGER | 最近更新时间 epoch millis | NOT NULL |

2 个索引：

- `index_llm_endpoints_orderIndex`（按排序链遍历端点）
- `index_llm_endpoints_protocol`（按协议过滤）

无外键。**API Key 不入此表**（Room 不加密），仍走 EncryptedSharedPreferences，key 改为 `api_key_ep_<endpointId>`（见 [16 配置与密钥管理](./16-配置与密钥管理.md)）。

`LlmEndpointDao` 提供端点 CRUD + `shiftAllOrderIndex(offset)` + `setOrderIndex(id, newOrder)` 用于重排。
`ConfigRepository.reorderEndpoints(orderedIds)` 用 `database.withTransaction` 把 shift + N 次 setOrderIndex 包成原子操作。

MIGRATION_6_7 完整 DDL 摘要（详见 `core-data/.../di/DataModule.kt`）：

- `user_action_events`：4 索引（`occurredAt`、`(type, occurredAt)`、`session`、`(targetId, type)`）

迁移期间不依赖虚拟账号体系，无孤儿行过滤需求；6 张表为全新建表，无数据回填。

MIGRATION_7_8 完整 DDL 摘要（详见 `core-data/.../di/DataModule.kt`）：

- `llm_endpoints`：2 索引（`orderIndex`、`protocol`），无外键。仅建表 + 索引，端点数据由 `migrateLegacyProviderConfigsIfNeeded` 在首次访问时按需迁移（幂等，第二次直接走 `if (migrated.value) return`）。

Note：schema JSON 目录现有 `1.json` ~ `5.json`，缺 `6.json` / `7.json` / `8.json`。`AppDatabase` 声明 `exportSchema=true`，`6.json` / `7.json` / `8.json` 应由 Room 编译期自动导出，但开发环境无 Android SDK 无法运行 `./gradlew compileDebugKotlin` 触发导出。CI workflow（`.github/workflows/build-and-release.yml`）仅做 `compileDebugKotlin`，无 Room schema 校验步骤，故不阻塞 CI。建议在有 Android SDK 的本地环境跑一次 `./gradlew compileDebugKotlin`，将自动生成的 `6.json` / `7.json` / `8.json` 一并提交，或后续在 CI 增加 `validateMigration` 步骤。

## TypeConverters

- Json 配置 `ignoreUnknownKeys=true`、`encodeDefaults=true`。
- 4 组：
  - `List<String>` <-> `String`（JSON 数组，null -> `"[]"`）
  - `List<Boolean>` <-> `String`（`activeWindows` 24 槽）
  - `InteractionType` <-> `String?`（枚举 `name`，`valueOf` + `runCatching` 容错 null）

## 关键查询

- `getActiveInHour(hour)`：JOIN `account_active_hours` `WHERE isVirtual=1 AND hour=:hour`（IMPL-38 性能优化，避免全表+JSON 反序列化）。
- `getFeedPagingSource`：Paging 3 数据源 `ORDER BY createdAt DESC`。
- `countByAuthorInWindow(authorId, windowStart, windowEnd)`：`[windowStart, windowEnd)` 区间计数（P1 修复窗满判断）。
- `countByAuthorSince(authorId, startOfDay)`：每日配额检查。
- `getPendingBefore(time)`：`WHERE scheduledAt<=:time AND executedAt IS NULL ORDER BY scheduledAt ASC`。
- `getCommentsForTweet`：LEFT JOIN `accounts`，作者被删时字段 null 兜底。
- `getCallStatistics`：`COALESCE(SUM,...)` + 互斥分类（`rate_limited` 优先），P2 修复。

### 用户行为建模相关查询（#146 新增）

- `UserActionDao.queryBetween(start, end)`：`WHERE occurredAt BETWEEN :start AND :end ORDER BY occurredAt ASC`，基础分析层拉取事件窗口。
- `UserActionDao.queryByType(type, since)`：按类型过滤事件。
- `UserActionDao.countSince(time)`：触发器判断是否达到 `COUNT_THRESHOLD=100`。
- `UserActionDao.queryScenarioEventsSince(scenarioId, since)`：A/B 反哺效果回测拉取 driven/control 标记事件。
- `UserActionDao.updateExtra(id, extra)`：`EventTextPreParser` LLM 预解析后写回 extra。
- `UserActionDao.deleteBefore(cutoff)`：`EventCleanupWorker` 周期清理过期事件。
- `UserProfileDao.setActive(@Transaction, versionId)`：原子切换激活版本（旧 active 置 0 + 指定版本置 1）。
- `UserProfileDao.deleteOldestInactive(limit)`：超 `MAX_PROFILE_VERSIONS=100` 时清理最旧非激活版本。
- `UserProfileDao.versionsStrictlyBeforeTime(time)`：`FeedbackAction.RollbackProfileVersion.aroundTimestamp` 定位用。
- `UserProfileOverrideDao.markSupersededAndInsert(@Transaction, type, key, ...)`：override 软删除 + 新覆盖插入原子事务。
- `UserProfileOverrideDao.active()`：`WHERE superseded=0` 过滤有效覆盖。
- `UserProfileFeedbackDao.recent(limit)` / `countSince(time)` / `countByRoleSince(role, time)`：智能体上下文 + 限流计数。
- `UserProfileRollbackDao.all()`：版本回滚审计时间线。

## 设计要点

- `@Upsert` vs `@Insert(REPLACE)`：引入 CASCADE 后 REPLACE 会先 DELETE 触发级联删除子表，故 upsert 用 `@Upsert`。
- 反向索引表 `account_active_hours`：核心性能优化。
- 原子计数 `executeInteractionsAndUpdateTweet`：`markExecuted` 返回值作幂等守卫避免重试翻倍。
- 双写一致性 `updateDynamicFields`：单事务内同时写 `PersonaDynamicFieldEntity` 与 `AccountEntity` denormalized 摘要。
- 孤儿行过滤：MIGRATION_3_4 用 `WHERE EXISTS` 过滤无父行的数据。
- 用户行为建模 6 张表无外键：用户行为数据独立于虚拟账号体系，不随账号级联删除，仅受"清除我的画像数据"按钮手动清空。
- override 软删除 + 审计链：`markSupersededAndInsert` `@Transaction` 保证"旧覆盖置 superseded=1 + 新覆盖插入"原子；旧覆盖不删除，可追溯覆盖历史；`resetAll()` 才物理删除。
- 版本激活原子性：`UserProfileDao.setActive` `@Transaction` 保证同一时刻至多一个 `isActive=1`，避免双 active。
- 事件 TTL 清理：`EventCleanupWorker` 24h 周期，读 `ConfigRepository.getEventTtlDays`（默认 14，`coerceAtLeast(1)`）后 `deleteBefore(now - ttlDays)`，无网络约束，避免事件表无限膨胀。
